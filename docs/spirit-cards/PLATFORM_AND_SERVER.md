# 霊宝コレクション（仮）— サーバー & 配信プラットフォーム メモ v0.1

> 「サーバーのキャパ・値段・技術」と「App Store / Steam に出す場合の差分」を1枚にまとめた検討メモ。
> 前提：**課金要素は作らない**（無課金）。設計の核は [GAME_DESIGN.md](GAME_DESIGN.md) の有限供給(INV-1〜6)。
> 価格は概算・契約前に各社最新を要確認。実装はまだ着手していない（検討段階）。

---

## 0. 大前提：負荷が性質の違う2つに分かれる

| ワークロード | 中身 | 重さ | 必要な性質 |
|---|---|---|---|
| **A. リアルタイム共有世界** | Colyseus `mmo`：位置同期30tick・mob・戦闘 | 重い（CPU・帯域） | 常時起動・低遅延・状態をメモリ保持 |
| **B. カード経済** | 在庫/所有/取引のDB、認証 | 軽い（だが厳密） | 耐久性・ACIDトランザクション（保存則INV-1〜6） |

- コストとスケールを決めるのは圧倒的に **A**。**B は数万ユーザーでも数百MB級**で、安いマネージドで足りる。
- 「サーバーを変える」議論は実質 **A をどこで動かすか**。
- **重要な設計判断**：**DBを唯一の真実源**にし、探索の払い出し・取引のスワップを**DBトランザクションで原子的に**行う。そうすれば server は使い捨て可能になり、サーバー乗り換え・複数プロセス化でも核(有限供給)が壊れない。

## 1. 現状のMMO資産（unspottable-web）

- スタック：client = Phaser 3 + TS + Vite、server = Node + **Colyseus 0.16**。**永続化レイヤーはゼロ**（DB/認証なし）。
- ルーム3種（`server/src/index.ts`）：`unspottable`・`bomberman`（`filterBy(["code"])`）、**`mmo`（filterByなし＝単一共有ワールド, maxClients=16）**。
- `mmo` は既に「歩き回る共有世界」（移動・叩く・mob24体・EXP/レベル/HP/ATK）。ただし **state はインメモリで再起動で消滅**、プレイヤーは `id=sessionId` の一時的存在。
- → 霊宝ゲームの器として最適だが、**核を成立させるには〈恒久アカウント＋耐久ストレージ〉の土台が前提**。

## 2. キャパの目安（Colyseusリアルタイム = ワークロードA）

- 30tickで16人＋mob24体 → 1クライアント下り **〜20-30 KB/s**、1ルーム合計 **〜3-4 Mbps**。CPUは軽い。
- ボトルネックは **CPU（状態シリアライズ/ブロードキャスト）と帯域**。
- 効くのは登録数でなく **同時接続数(CCU)**。収集ゲームはCCUが低めに出やすい（登録1万でもCCU数十〜数百のことが多い）。

| 規模(CCU) | スペック目安 | 月額レンジ |
|---|---|---|
| 〜20人（個人テスト） | 0.5 vCPU / 512MB-1GB | $0〜7 |
| 〜100人 | 1 vCPU / 1-2GB 常時起動 | $7〜25 |
| 〜500人 | 2-4 vCPU（複数プロセス）＋LB | $25〜80 |
| 1,000人超 | マルチノード＋DB真実源＋Redis | $80〜数百 |

ACID対応RDB推奨（保存則INVをTxに落とすため）。

データ量は小さく、**当面どの無料/最安枠でも足りる**。

## 6. 設計ガードレール（無課金で核を守るための3点）

このゲームの面白さ＝「有限供給×取引×ガチャ的探索」は規制で最も厳しく見られる組み合わせ。
**課金を入れない**ことで地雷の大半が外れるが、以下3点は配信先に関係なく守る：

1. **探索は無課金**（時間回復スタミナのみ。有料ガチャにしない）
2. **取引はカード↔カードの物々交換のみ**（リアルマネー換金・販売を作らない＝RMT/賭博規制の崖を回避）
3. **コンプ報酬は景品性なし**（称号等。🇯🇵コンプガチャ規制=景表法は“有料ガチャで揃えさせる”が対象なので、無課金＋景品性なしなら抵触しない）

> 課金で確率や希少カードに手を出した瞬間、確率開示・コンプガチャ・換金規制のレイヤーが一気に増える。今回は無課金なので安全圏。

## 7. 配信先A：App Store（iOS）

### ネイティブ化（器）

| 方式 | 中身 | 手間 | リスク |
|---|---|---|---|
| PWA→Capacitor包み | Web資産流用 | 小 | 「ただのWebサイト」判定で**リジェクト(4.2)**。native機能を足せば緩和 |
| **Expo / React Native**（推奨） | メタUIをRN化、世界は WebView/RN枠 | 中 | 試作Reactロジック移植しやすい。本命 |
| Unity/Godot 等 | フルネイティブ | 大 | 2D収集には過剰 |

### 無課金で効く差分

| 項目 | 課金ありなら | **無課金（今回）** |
|---|---|---|
| ガチャ確率開示(3.1.1) | 必須 | **対象外** |
| 🇯🇵コンプガチャ(景表法) | 抵触リスク | **ほぼ解消** |
| IAP手数料 15〜30% | 徴収 | **ゼロ** |
| IAPレシート検証 | 必要 | **不要** |
| 年齢レーティング | 上がりがち | **低く収まる** |

### 無課金でも残るもの

- **Apple Developer Program $99/年**（出品に必須・固定費）
- アプリ内**アカウント削除導線**・プライバシー表示（アカウントを持つ以上必須）
- **Sign in with Apple**：他SNSログインを出す場合のみ併設必須
- ネイティブビルドで「Webサイト」リジェクト回避
- プッシュ通知(APNs)：無料

### App Store版の推奨構成

```
[iOSアプリ Expo/RN]
   ├─ メタUI（台帳・探索・取引）… 非同期REST/DB中心
   ├─ Sign in with Apple
   └─ プッシュ通知（取引オファー）
                 ▼
[バックエンド] DB=唯一の真実源（在庫/所有/取引・INV）
   └─（任意）Colyseus＝歩き回る世界。v1は省略し非同期取引にすると常時接続代が消える
```

- モバイルは接続が不安定 → **リアルタイム世界をv1で省き、取引を非同期（オファー受信箱＋プッシュ）**にするとサーバー代も体験も有利。
- 固定費は実質 **$99/年 ＋ サーバー$0〜5/月**。

## 8. 配信先B：Steam（PC）

> 結論：**このゲームはむしろSteamの方が相性が良い面が多い**（PCはリアルタイム世界に耐性、取引/インベントリ基盤が公式に存在、長期コストが安い）。

### コスト・審査

| 項目 | Steam | App Store比較 |
|---|---|---|
| 出品料 | **Steam Direct $100（アプリ毎・1回きり）** ※売上$1000で還元、無料配布なら実質$100の一括負担 | Appleは**$99/年（毎年）** → 長期はSteamが安い |
| プラットフォーム手数料 | 売上の30%（$10M超で25%…） | 無課金なら**両方とも無関係** |
| 審査 | **軽い**（起動確認・マルウェア・コンテンツ申告中心）。「Webラッパー禁止」のような厳格さは無い | Appleは厳しめ |
| 対応OS | Windows/Mac/Linux（**デスクトップ**） | iOSモバイル |

### 器（ネイティブ化）

- Phaser(Web)を **Electron / Tauri でデスクトップ実行ファイル化** → Steamに載せられる。Apple的な「Webサイト拒否」が無く、**既存Web資産をほぼそのまま流用しやすい**。
- 入力はマウス/キーボード/コントローラ。**PCプレイヤーは常時接続セッションに耐性がある** → 「歩き回る共有世界(Colyseus)」をv1から入れる現実味がモバイルより高い。

### 認証・ネットワーク（Steamworks SDK・全て無料）

- **Steam ログイン**（Steamworks/OpenID）でアカウント。Sign in with Apple 相当が無料で手に入る。
- **Steam Datagram Relay (SDR)**：Valveのリレー網経由でP2P/サーバー通信。DDoS耐性・NAT越えを肩代わり。
- **Steam Game Servers / マッチメイキング**：利用可能。ただし**カード在庫の真実源となるDBは自分で持つ**点はApp Storeと同じ（Steamはゲームロジックをホストしない）。
- **Steam Cloud**：セーブ同期（無料）。ただし権威データはサーバー側なので補助的。

### ★ Steam Inventory / Trading（このゲームの核と直結）

- Steamには **Steam Inventory Service** と **Steam Trading（プレイヤー間アイテム交換）** が**公式機能として存在**。CS/TF2のアイテム交換と同じ仕組み。**カード収集×物々交換ゲームにほぼ専用設計**。
- これを使えば交換UI/所有管理の一部を肩代わりできる可能性。ただし：
  - **Steam Community Market（実マネー売買）に乗せると、Steam Wallet＝リアルマネーが絡み、RMT/賭博規制（CS:GOロットボックス問題の系譜）に逆戻り**。
  - → **使うなら「Steam Trading のプレイヤー間物々交換」だけ**にとどめ、Marketへの上場・換金は作らない（§6ガードレールと同じ）。
  - 有料ランダムアイテム（ルートボックス）には**確率開示義務**があるが、**無課金なら対象外**。
- 注意：Steam Inventory に乗せると「在庫の真実源」がSteam側にも分散する。**核(保存則INV)を自前DBで一元管理したいなら、Steam Inventoryは使わず自前DB＋自前交換UIにする**方が設計が綺麗（要判断, §10）。

### Steam版の推奨構成

```
[PCアプリ Electron/Tauri + Phaser]
   ├─ Steam ログイン（Steamworks）
   ├─ （任意）Steam Datagram Relay でネット越え
   └─ リアルタイム世界(Colyseus)をv1から入れやすい
                 ▼
[バックエンド] DB=唯一の真実源（在庫/所有/取引・INV）  ← App Store版と共通
```

- 固定費は **$100（1回）＋ サーバー$5〜30/月**。毎年課金が無いぶん**長期はApp Storeより安い**。

## 9. App Store vs Steam 早見表（無課金前提）

| 観点 | App Store(iOS) | Steam(PC) |
|---|---|---|
| 出品費 | $99/**年** | $100/**1回** |
| 審査の厳しさ | 厳しい | 緩い |
| Webラッパー許容 | 渋い(4.2) | 寛容（Electron可） |
| リアルタイム世界 | モバイルは不安定→非同期推奨 | PCは耐性あり→v1から可 |
| 取引基盤 | 自前で実装 | **公式Trading/Inventoryあり**（使うかは要判断） |
| 認証 | Sign in with Apple | Steamログイン（無料） |
| 既存Web資産の流用 | RN移植が要る | **そのまま近い形で流用しやすい** |
| 観客層 | モバイルのカジュアル大衆 | PCゲーマー・収集/交換文化が濃い |
| 既存unspottable-web資産 | 一部移植 | **Phaser/Colyseusをほぼ活かせる** |

> ざっくり：**手早く既存資産を活かし、リアルタイム世界も入れたいなら Steam**。**スマホの大衆リーチが欲しいなら App Store（その場合は非同期取引でサーバー軽量化）**。バックエンド(DB=真実源)は両者共通なので、**先にバックエンドを固めれば後から配信先を選べる/両対応できる**。
