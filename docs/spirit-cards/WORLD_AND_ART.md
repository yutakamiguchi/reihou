# 霊宝コレクション — 世界観 & アートディレクション（画像生成用）

> ChatGPT等で画像生成する際に、この文書の「世界観」「画風」「素材仕様」をそのまま渡す。
> 命名・配置の技術ルールは [../../.claude](character-art-pipeline メモリ) と本書を併用。

---

## 1. 世界観（コピペ用）

```
舞台は「失われた霊宝（れいほう）を集める世界」。
かつて世界には、精霊・自然・古き獣・天体・遺物に宿った“霊宝”が満ちていた。
だが大いなる厄災で霊宝は砕け散り、各地に失われた。
探索者たちは魔物の徘徊する広野をさまよい、霊宝を見つけて集め、
互いに交換しながら、世界に数えるほどしか残らぬ伝説の「秘宝」を追い求める。

トーン：神秘的・幻想的で、少し物悲しくも温かい。和洋折衷のファンタジー。
HUNTER×HUNTERのグリードアイランド的な「収集と交換」の高揚感が根底にある（IPは使わずオリジナル）。
```

## 2. 画風（コピペ用）

```
画風：手描き風のやわらかな水彩＋繊細な線画。淡い発光、神話的で上品。
過度に派手・けばけばしくせず、落ち着いた彩度。
レアリティで色味を分ける：
  普通(common)：くすんだ青灰（#8a93a8）
  希少(rare)：澄んだ水青（#5fb6c4）
  秘宝(legend)：荘厳な黄金（#e8b04b）＋淡いオーラ
背景の基調は深い紫紺（#15101f）。
```

## 3. 霊宝カード絵の仕様

- **1枚＝1つの物/存在**を中央に配置。正方形寄り。
- **背景は透過PNG**（または上記の深い紫紺の単色）。**文字・枠・ラベルは入れない**（UIで付与）。
- レアリティで格を表現：普通は素朴、希少は美しく、秘宝は神々しく（光・装飾）。
- 系統テーマ：自然 / 獣 / 遺物 / 天体 / 精霊 / 災厄。

### 50種リスト（季節「原初」）
普通(26): 灯火の精・川の囁き・苔むす石・風の使い・夜鳴鳥・古びた鍵・旅人の靴・銅の護符・朝露の珠・野ねずみ・焚き火の炭・砂時計・錆びた剣・蝋燭・木の実・釣り針・ランタン・古びた巻物・銀の匙・兎の足・石畳のかけら・麦の穂・古いコイン・羽根ペン・土笛・ひび割れた小瓶

## 4. 敵（魔物）画像の仕様

- ボンバーマンのキャラ同様、**透過PNG**・文字なし。正面(front)1枚でOK（凝るなら front/side/back）。
- 5種、それぞれ色・シルエットで強さが伝わるように：

| 種別 | 名前 | 印象 | 色 |
|---|---|---|---|
| grunt | 迷い霊 | 小さく漂う弱い霊魂 | 淡い緑 |
| swift | 疾風鬼 | 風をまとう俊敏な小鬼/精 | 水色 |
| tank | 岩石巨人 | 苔むした鈍重で頑丈な石の巨人 | 茶 |
| brute | 紅蓮鬼 | 炎を纏う攻撃的な赤鬼 | 赤 |
| boss | 災厄の主 | 厄災の化身。巨大で禍々しく威厳 | 紫 |

## 5. 技術ルール（保存時）

- 透過PNG・ラベル文字なし・サイズ不揃いOK（表示はコードで調整）。
- 霊宝カード：置き場は別途決定（例 `client/public/relics/<id>.png`）。今はUIで絵文字アイコン。
- 敵：`client/public/char-bomberman/` と同様、専用フォルダに front 等で。
